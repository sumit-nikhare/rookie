Roles and Responsibilities

As the lead designer for the TARP Design System, I owned the end-to-end process of building a scalable and cohesive UI foundation. My responsibilities spanned strategic planning, system architecture, visual design, and cross-functional collaboration.

  • Defined the design system architecture, including components, patterns, layout rules, spacing, and interaction guidelines.
  • Created a modular component library in Figma, ensuring consistency, reusability, and responsiveness across web and mobile breakpoints.
  • Established a visual language, including typography, color schemes, iconography, and motion principles aligned with the product's branding.
  • Collaborated closely with developers to ensure pixel-perfect implementation through design specs, component documentation, and regular syncs.
  • Facilitated stakeholder alignment by presenting design rationale and demonstrating the system's value through real-world use cases.
  • Maintained and evolved the system by setting up documentation, usage guidelines, and feedback loops for continuous improvement.
  • Advocated for design consistency, ensuring that all modules—campaign management, CRM, FAQs, analytics—followed the same UI/UX principles.

Problem Statement

Context & Background

Permute Solutions Pvt. Ltd., a startup focused on creating agile software solutions, recognized the need for a unified design system to accelerate development and ensure consistency across their product ecosystem. The company was building multiple applications with varying UI patterns, leading to development inefficiencies and inconsistent user experiences.

  • Development teams were creating UI components from scratch for each project, resulting in inconsistent design patterns and increased development time.
  • Multiple applications within the company ecosystem lacked visual cohesion, creating a fragmented user experience.
  • Design decisions were being made in isolation, leading to maintenance challenges and scalability issues.
  • There was no centralized design language or component library to guide developers and designers.
  • Permute Solutions needed a scalable design system that could support rapid prototyping and development while maintaining design quality and consistency.

User Needs & Business Goals

The TARP Design System was created to serve both the internal development teams and the end users of Permute Solutions' applications. The system needed to balance developer efficiency with user experience quality while supporting the company's agile development methodology.

Developer & Designer Needs:

  • A comprehensive component library with reusable UI elements to accelerate development cycles.
  • Clear design guidelines and documentation to ensure consistent implementation across teams.
  • Responsive design patterns that work across different screen sizes and devices.
  • A flexible grid system that supports various layout requirements and content types.

Business Goals:

  • Reduce development time and costs by providing pre-built, tested UI components.
  • Establish a consistent brand identity across all company products and applications.
  • Improve user experience quality through standardized, well-tested design patterns.
  • Enable faster prototyping and iteration cycles to support agile development methodology.

UI Designs/Screens

A series of high-fidelity UI screens demonstrating the application of the TARP Design System across real-world use cases. From data dashboards to campaign setup and help centers, each screen reflects a balance of usability, visual clarity, and system-wide consistency.

Screen 1 - Knowledge Centre

This screen showcases a centralized knowledge hub where users can browse articles and FAQs in a clean, card-based layout. The left section features visually distinct article previews categorized by topic (e.g., Education, Technology), while the right section provides instant Q&A results based on user queries. Designed for clarity and quick discovery, this screen enhances learning and onboarding experiences through effective use of hierarchy, tags, and visual grouping.

Screen 2 – Campaign Profile Form

This screen allows users to complete or edit campaign profiles through a structured, multi-section form. The layout includes fields for campaign type, use cases, date ranges, and scripts, all grouped with clear labels and intuitive spacing. The right-hand sidebar features a dynamic "Help Centre" prompt for user assistance, improving support accessibility. Designed with a focus on clarity and task flow, this screen ensures that users can input detailed data without cognitive overload.

Screen 3 – Performance Dashboard & Scheduled List

This dashboard offers a visual snapshot of performance metrics alongside a detailed scheduled activity list. The top section presents a target summary using clear data visualizations—percentages, ratios, and status indicators—for fast, at-a-glance insights. Below, the Scheduled List organizes user interactions (calls, SMS, WhatsApp) by partner, location, language, and milestone. The design emphasizes clarity, using soft colors, icon tags, and visual separation to improve scan-ability and support daily operations with minimal friction.

Screen 4 – Finance Dashboard & Campaign Setup

This screen integrates financial insights with campaign setup functionality. The top panels display key wallet, card, and budget data using clear numeric indicators and color-coded trends. Circular charts provide a quick breakdown of expenditures and budget allocations. Below, users can launch short surveys or campaigns with a structured form, complete with campaign type, dates, and custom scripts. The right-hand Expenditure List offers a summarized view of partner interactions. This unified layout enhances visibility into both financial performance and campaign operations—helping users make data-driven decisions with ease.

Screen 5 – FAQ Explorer & Topic Navigation

This screen showcases a structured FAQ interface under the "Knowledge Centre." On the left, a collapsible table of contents allows users to browse categorized topics like EC2, Container Service, and Kubernetes. On the right, an accordion-style question-answer module provides quick, expandable access to detailed information. The active question is highlighted for focus and readability. Designed for self-service support and contextual learning, this layout improves knowledge discovery through clean interaction patterns and a distraction-free reading experience.
